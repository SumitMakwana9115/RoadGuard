## ⚡ The Vision
**RoadGuard** isn't just a complaint system; it's a high-performance urban intervention engine. Designed for the modern citizen and the efficient city administrator, it bridges the gap between infrastructure decay and rapid resolution with surgical precision.

Built for the **University Submission (Enrollment: 230210107035)**, this system demonstrates how technical rigor meets state-of-the-art UI/UX design.

---

## 🚀 Key Features

### 🏢 Area-Based Hierarchy
A multi-tier geographical model ensuring no pothole goes unmapped.
- **Ward** → **Area** → **Spot**
- Granular tracking for precise maintenance dispatch.

### ⏱️ SLA-Driven Workflow
Time is the ultimate metric. RoadGuard enforces strict SLAs:
- **7 Hours**: Initial Response Deadline.
- **48 Hours**: Complete Resolution Deadline.
- Automated escalation for breaches.

### 🔐 Triple-Role Security
- **Complainant**: File, track, and reopen complaints.
- **Staff**: Take action, upload proof, and resolve.
- **Supervisor**: Audit, assign, and verify.

### 📊 Real-time Intelligence
Dynamic dashboards providing a "Bird's Eye View" of the city's health.

## 🛠️ Technical Stack

- **Backend**: PHP 7.4+ with strict modular architecture.
- **Database**: MySQL 8.0 with optimized indexing and foreign key constraints.
- **Frontend**: Vanilla CSS3 (Custom Variables, Flex/Grid) & ES6+ JavaScript.
- **Security**: Prepared statements, session protection, and role-based access control (RBAC).

## 📥 Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/SumitMakwana9115/RoadGuard.git
   ```
2. **Setup Database**:
   - Import `database.sql` into your MySQL server.
3. **Configure**:
   - Update `config/db.php` with your credentials.
4. **Deploy**:
   - Move to your `htdocs` or equivalent web root.
   - Access via `http://localhost/WP`.
